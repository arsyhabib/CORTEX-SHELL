## 🔄 CONTENT GENERATION WORKFLOW

### Phase 1: Prepare Module Structure (Day 1)
**Input:** Domain knowledge (medical/learning content)  
**Output:** Module metadata + outline  
**Duration:** 2-3 hours

1. **Create Module Metadata**
   ```json
   {
     "module_id": "mod_neuro_stroke_001",
     "title": "Acute Ischemic Stroke",
     "subtitle": "Patofisiologi, Diagnosis, dan Manajemen",
     "domain": "neuro_emergency",
     "author": "Dr. [Name]",
     "year": 2025,
     "batch_id": "batch_1",
     "estimated_sections": 10,
     "estimated_hours": 3,
     "learning_goals": [
       "Menjelaskan patofisiologi stroke iskemik...",
       "Membedakan jenis-jenis stroke...",
       "Menganalisis temuan klinis dan diagnostic..."
     ]
   }
   ```

2. **Create Section Outline**
   - Define 8-15 sections with headings
   - Estimate time per section (15-30 min each)
   - Map visuals/assets needed per section

### Phase 2: Generate Content Using PROMPT_MATERI_REFINED (Day 2-3)
**Input:** Module outline + source materials  
**Output:** Complete markdown with Grade 1-4  
**Tool:** Use PROMPT_MATERI_REFINED.md

**Processing Steps:**
1. Extract from source material (PPT, PDF, slides)
2. Create Grade 1-4 information hierarchy
3. Structure 6-15 expanded sections
4. Format with bold, italic, underline for emphasis
5. Create glossary (min 5 terms)
6. List clinical references + exam focus
7. Validate all fields complete

**Expected Output Size:**
- Per module: 3,000-5,000 words
- Per section: 250-400 words
- Includes: overview (300 words), 10-12 sections (300-400 each), glossary (5-10 terms), callouts (15-20)

### Phase 3: Parse Markdown to JSON (Day 3)
**Tool:** Node.js parser script (to be created)  
**Input:** content.md files  
**Output:** JSON structures for Surfaces 2-4

**Parsing Logic:**
1. Extract metadata section → `metadata.json`
2. Parse "OVERVIEW" → Surface 3 module_overview
3. Parse "LEARNING GOALS" → Surface 3 learning_goals array
4. Parse each "EXPANDED SECTIONS" → Individual section JSON objects
5. Extract content_blocks → Surface 4 detail_panel content
6. Extract glossary → Surface 14 terms (Batch 2)
7. Extract callouts → Surface 11 pearls (Batch 2)
8. Generate search index entries for Surface 6

### Phase 4: Generate Search Index (Day 4)
**Tool:** Index generation script  
**Input:** All parsed content from Phase 3  
**Output:** `app_data/system/search/search_index.json`

**Index Content:**
- All module titles + descriptions
- All section headings + keywords
- All glossary terms
- Tagging: domain, content_type, difficulty, module_id

### Phase 5: Validate & QA (Day 5)
**Checklist:**
- [ ] All modules have complete metadata
- [ ] All sections have: heading, subheading, 3+ content blocks
- [ ] Surface 2 displays hero + module preview
- [ ] Surface 3 shows full module outline + goals
- [ ] Surface 4 renders section detail with assets
- [ ] Search index contains all searchable items
- [ ] No broken references or missing fields
- [ ] Word counts align with specification
