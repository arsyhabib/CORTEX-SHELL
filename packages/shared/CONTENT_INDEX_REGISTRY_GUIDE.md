# CONTENT INDEX REGISTRY GUIDE — content_index.kimi.json
**Version:** 2.0  
**Package:** packages/shared  
**File managed:** app_data/runtime/content_index.kimi.json

---

## OVERVIEW

`content_index.kimi.json` is the central registry for all content in CORTEX-SHELL. The app reads this file at runtime to discover:
- All lecture domains and their modules
- All exam sets
- Question counts for UI rendering

This file must be kept accurate and valid at all times. Any content that is not registered here will not appear in the app.

The file lives at:
```
app_data/runtime/content_index.kimi.json
```

---

## FILE STRUCTURE OVERVIEW

```json
{
  "generatedAt": "2026-06-10",
  "lectureDomains": [...],
  "lectureModuleCount": 2,
  "examSets": [...],
  "examSetCount": 2,
  "examQuestionCount": 5
}
```

### Top-Level Fields

| Field | Type | Description |
|---|---|---|
| `generatedAt` | string (ISO date) | Date this file was last updated. Update to today's date every time the file is modified. Format: YYYY-MM-DD. |
| `lectureDomains` | array | List of lecture domain objects. Each domain contains one or more modules. |
| `lectureModuleCount` | integer | Total number of modules across all domains. Must be manually recalculated after any addition. |
| `examSets` | array | List of exam set objects (both exam_asli and bank_soal_ai). |
| `examSetCount` | integer | Total number of entries in `examSets[]`. |
| `examQuestionCount` | integer | Sum of all `questionCount` values across all exam sets. |

## ADDING A NEW LECTURE DOMAIN

A lecture domain groups related modules under a single subject area.

### When to Add a New Domain

Add a new `lectureDomains[]` entry when:
- A new medical subject or block is being onboarded (e.g., "Kardiologi", "Respirologi")
- The new modules do not fit logically under any existing domain

### Domain Object Schema

```json
{
  "id": "cardiology_acute",
  "title": "Kardiologi Klinis — Sindrom Koroner Akut",
  "summary": "Modul klinis kardiologi: ACS, STEMI, NSTEMI, dan tata laksana antiplatelet.",
  "moduleCount": 0,
  "modules": []
}
```

| Field | Required | Notes |
|---|---|---|
| `id` | Yes | Unique domain identifier. Use snake_case. Pattern: `[subject]_[qualifier]`. |
| `title` | Yes | Human-readable title. Use bilingual format: English clinical name + Indonesian subtitle where applicable. |
| `summary` | Yes | 1–2 sentence description of the domain scope. |
| `moduleCount` | Yes | Count of modules in the `modules[]` array. Start at 0 for empty domains. |
| `modules` | Yes | Array of module objects. Start as empty array `[]`. |

### Step-by-Step: Add a New Domain

1. Open `content_index.kimi.json`
2. Add a new object to the `lectureDomains[]` array (append at end or insert in logical order)
3. Set `moduleCount: 0` and `modules: []` initially
4. Update `generatedAt` to today's date
5. Validate JSON syntax before saving

## ADDING A NEW MODULE TO AN EXISTING DOMAIN

### When to Add a Module

Add a module entry after:
- The `module.json` file has been generated by the AI agent
- The `module.json` has been validated and placed at its output path
- The `content.md` source document exists

### Module Object Schema

```json
{
  "id": "mod_cardio_acs_001",
  "title": "Acute Coronary Syndrome: Pathophysiology, Diagnosis & Management",
  "sourceFile": "app_data/domains/materi/modules/mod_cardio_acs_001/module.json",
  "sourceDocument": "app_data/domains/materi/modules/mod_cardio_acs_001/content.md",
  "topicTags": ["ACS", "STEMI", "NSTEMI", "Antiplatelet", "Kardiologi"],
  "status": "active",
  "domainId": "cardiology_acute",
  "order": 1,
  "batchId": "batch_1",
  "estimatedTimeMinutes": 180,
  "sectionCount": 10,
  "wordCount": 7200
}
```

| Field | Required | Notes |
|---|---|---|
| `id` | Yes | Unique module ID. Pattern: `mod_[subject_short]_[topic]_[NN]`. Must be globally unique across all domains. |
| `title` | Yes | Full module title. Use English for clinical/academic titles. |
| `sourceFile` | Yes | Path to `module.json`, relative to project root. No leading slash. |
| `sourceDocument` | Yes | Path to source content document (`.md` file), relative to project root. |
| `topicTags` | Yes | 3–6 topic tags. Use both English and Indonesian terms for searchability. |
| `status` | Yes | See Status Values section below. |
| `domainId` | Yes | Must match the `id` of the parent domain in `lectureDomains[]`. |
| `order` | Yes | Display order within the domain (integer, starting at 1). |
| `batchId` | No | Batch ID from the generation process. |
| `estimatedTimeMinutes` | No | Estimated study time in minutes. |
| `sectionCount` | No | Number of sections in the module. |
| `wordCount` | No | Approximate word count of the module content. |

### Step-by-Step: Add a Module to an Existing Domain

1. Locate the target domain object in `lectureDomains[]`
2. Append the new module object to the domain's `modules[]` array
3. Increment the domain's `moduleCount` by 1
4. Increment the top-level `lectureModuleCount` by 1
5. Update `generatedAt` to today's date
6. Validate JSON syntax before saving

## ADDING A NEW EXAM SET

### When to Add an Exam Set

Add an exam set entry after:
- The `exam_set.json` has been produced by the Vision Agent
- The file has passed all checks in `OUTPUT_VALIDATION_CHECKLIST.md`
- The file is placed at its canonical path

### examSets[] Entry Schema

```json
{
  "id": "set_asli_02",
  "title": "Ujian Akhir Blok Neurologi 2022",
  "type": "asli",
  "questionCount": 90,
  "sourceFile": "app_data/domains/exam_asli/exam_sets/set_asli_02/exam_set.json",
  "source_label": "Univ Neurologi — Ujian Akhir 2022",
  "status": "active",
  "year": "2022",
  "subject": "Neurologi Klinis",
  "batchId": "exam_asli_batch_02"
}
```

| Field | Required | Notes |
|---|---|---|
| `id` | Yes | Must exactly match `exam_id` in the exam_set.json file. |
| `title` | Yes | Human-readable title for the exam set. |
| `type` | Yes | `"asli"` or `"ai_generated"`. Must match the `type` field in exam_set.json. |
| `questionCount` | Yes | Must match `metadata.total_questions` in exam_set.json. |
| `sourceFile` | Yes | Path relative to project root. No leading slash. No trailing slash. |
| `source_label` | Yes | Short label shown in the app UI (e.g., institution name + exam type). |
| `status` | Yes | See Status Values section below. |
| `year` | No | Exam year. String. |
| `subject` | No | Medical subject/block. |
| `batchId` | No | Processing batch ID. |

### Step-by-Step: Add an Exam Set

1. Append a new object to the `examSets[]` array
2. Increment `examSetCount` by 1
3. Add `questionCount` to the running total in `examQuestionCount`
4. Update `generatedAt` to today's date
5. Validate JSON syntax before saving

### Updating examQuestionCount

`examQuestionCount` must always equal the sum of all `questionCount` values in `examSets[]`.

When adding a new exam set with 90 questions:
```
examQuestionCount (new) = examQuestionCount (old) + 90
```

After any modification to `examSets[]`, recalculate from scratch to catch any drift:
```
examQuestionCount = sum(examSets[].questionCount)
```

## STATUS VALUES

Both modules and exam sets use the same status vocabulary:

| Status | Meaning | Use When |
|---|---|---|
| `"active"` | Published and visible in the app | Content is validated and ready for students |
| `"draft"` | In progress or under review | Content is generated but not yet validated |
| `"archived"` | Retired from active use | Superseded by newer content or withdrawn exam |
| `"Draft_extracted"` | Raw extraction complete, not yet reviewed | Agent has produced output, human review pending |

For new exam sets, use `"draft"` until the validation checklist is complete, then switch to `"active"`.

## SOURCE FILE PATH CONVENTIONS

All `sourceFile` paths must follow these rules without exception:

1. **Relative to project root** — The path is resolved relative to `/home/user/CORTEX-SHELL/`. Do not include the root directory itself.

2. **No leading slash** — The path must NOT start with `/` or `./`.

3. **Forward slashes only** — Use `/` as directory separator. Do not use `\`.

4. **Point to the file, not the directory** — The path must end with the actual filename (e.g., `module.json` or `exam_set.json`).

5. **Consistent depth** — Modules live at:
   ```
   app_data/domains/materi/modules/[module_id]/module.json
   ```
   Exam sets live at:
   ```
   app_data/domains/exam_asli/exam_sets/[exam_id]/exam_set.json
   app_data/domains/bank_soal_ai/question_sets/[exam_id]/exam_set.json
   ```
   Demo files live at:
   ```
   app_data/demo/[filename].json
   ```

## VALIDATION BEFORE SAVING

Before saving any modification to `content_index.kimi.json`:

1. **JSON syntax check** — Parse the entire file. Fix any trailing commas, unclosed brackets, or unescaped characters.

2. **Counter accuracy check** — Verify:
   - `lectureModuleCount` == total module entries across all domains
   - `examSetCount` == `examSets[].length`
   - `examQuestionCount` == sum of all `examSets[].questionCount`

3. **ID uniqueness check** — Verify no two modules share the same `id`, and no two exam sets share the same `id`.

4. **sourceFile existence check** — For every `sourceFile` path, verify the file exists on disk before saving.

5. **generatedAt updated** — The `generatedAt` field must reflect today's date.

## EXAMPLE: COMPLETE FILE AFTER ADDING ONE EXAM SET

Scenario: Adding `set_asli_02` with 45 questions to an existing registry.

Before:
```json
{
  "generatedAt": "2026-06-09",
  "lectureDomains": [...],
  "lectureModuleCount": 1,
  "examSets": [
    { "id": "set_asli_01", "questionCount": 2, ... }
  ],
  "examSetCount": 1,
  "examQuestionCount": 2
}
```

After:
```json
{
  "generatedAt": "2026-06-10",
  "lectureDomains": [...],
  "lectureModuleCount": 1,
  "examSets": [
    { "id": "set_asli_01", "questionCount": 2, ... },
    {
      "id": "set_asli_02",
      "title": "Ujian Akhir Blok Neurologi 2022",
      "type": "asli",
      "questionCount": 45,
      "sourceFile": "app_data/domains/exam_asli/exam_sets/set_asli_02/exam_set.json",
      "source_label": "Univ Neurologi — Ujian Akhir 2022",
      "status": "active"
    }
  ],
  "examSetCount": 2,
  "examQuestionCount": 47
}
```

Changes made:
- New entry appended to `examSets[]`
- `examSetCount`: 1 → 2
- `examQuestionCount`: 2 → 47 (2 + 45)
- `generatedAt`: updated to today
